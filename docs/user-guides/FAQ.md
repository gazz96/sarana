## üìù Problem Reporting

### **Q: How do I report a maintenance issue?**
**A:** 
1. Login to the system
2. Click **"Problems"** ‚Üí **"Create Problem"**
3. Describe the issue clearly
4. Select the location
5. Add affected items and estimated costs
6. Click **"Submit"**

### **Q: Can I save a problem report and finish it later?**
**A:** Yes! Click **"Save Draft"** instead of "Submit". Your draft will be saved and you can continue editing it later. Drafts have status "DRAFT" (0).

### **Q: What's the difference between "Save Draft" and "Submit"?**
**A:**
- **Save Draft**: Saves your work so you can continue later. The problem won't be processed yet.
- **Submit**: Sends your problem for review and starts the approval workflow.

### **Q: I made a mistake in my submitted problem. Can I edit it?**
**A:** Once submitted, you cannot edit directly. However:
- If still in **DRAFT** status: You can edit
- If **DIAJUKAN** or later: Contact admin to make changes

### **Q: How many items can I add to one problem report?**
**A:** You can add as many items as needed. There's no limit to the number of items per problem.

### **Q: Do I need to provide cost estimates?**
**A:** Yes, please provide realistic cost estimates for each item. This helps the finance team approve budgets faster. If unsure, provide a rough estimate.

### **Q: Should I upload photos of the damage?**
**A:** Yes! Photos help technicians understand the problem better and bring the right tools/parts. Upload clear photos showing the damage.

## üîÑ Problem Status & Workflow

### **Q: What do the different status codes mean?**
**A:**
- **0 - DRAFT**: Still being prepared
- **1 - DIAJUKAN**: Submitted for review
- **2 - PROSES**: Technician is working on it
- **3 - SELESAI DIKERJAKAN**: Repair completed
- **4 - DIBATALKAN**: Request cancelled
- **5 - MENUNGGU PERSETUJUAN MANAGEMENT**: Awaiting management approval
- **6 - MENUNGGU PERSETUJUAN ADMIN**: Awaiting admin approval
- **7 - MENUNGGU PERSETUJUAN KEUANGAN**: Awaiting finance approval

### **Q: How long does it take for a problem to be resolved?**
**A:** Resolution time varies by priority:
- **High Priority** (safety issues): Same day
- **Medium Priority**: 1-2 days
- **Low Priority**: 3-7 days

### **Q: My problem has been in the same status for days. What should I do?**
**A:**
1. Check if there are notes explaining the delay
2. Contact the assigned technician (if any)
3. If still unsure, contact your administrator

### **Q: Can I cancel a problem I've submitted?**
**A:** Yes, if it hasn't been processed yet. Contact your administrator to cancel the problem. Status will change to "DIBATALKAN" (4).

### **Q: Who approves my problem request?**
**A:** Approval workflow:
1. **Lembaga** (Management) reviews strategic importance
2. **Admin** checks technical feasibility
3. **Keuangan** (Finance) approves the budget

### **Q: How do I know when my problem is completed?**
**A:** You'll receive a notification when status changes to "SELESAI DIKERJAKAN" (3). You can also check "My Problems" to see the current status.

## üîß Technical Support

### **Q: The system is running slowly. What should I do?**
**A:**
1. Check your internet connection
2. Clear your browser cache
3. Try a different browser
4. If problems persist, contact technical support

### **Q: I can't upload photos. What's wrong?**
**A:** Check:
- File size is under 2MB
- File format is JPG or PNG
- Your internet connection is stable
- Browser permissions allow file uploads

### **Q: The page isn't loading correctly.**
**A:** Try these steps:
1. Refresh the page (F5 or Cmd+R)
2. Clear browser cache and cookies
3. Try a different browser
4. Check if it's a known system issue

### **Q: My data isn't saving. What should I do?**
**A:**
1. Check your internet connection
2. Make sure all required fields are filled
3. Try clicking "Save" again
4. Contact support if the issue persists

### **Q: What should I do if I don't have access to features I need?**
**A:** Check that you're logged in with the correct account. If you still lack access, contact your administrator to verify your role and permissions.

## üí∞ Budget & Finance

### **Q: How are repair costs approved?**
**A:** The approval process:
1. You submit the problem with cost estimates
2. Management reviews the necessity
3. Admin verifies technical details
4. Finance approves the budget
5. Work can begin after all approvals

### **Q: What if the actual repair cost is different from the estimate?**
**A:** Technicians will update the actual cost after completing the repair. If significantly different, it may require additional approval.

### **Q: How long does budget approval take?**
**A:** Typically 1-3 business days, depending on the amount and complexity. Urgent safety issues are prioritized.

### **Q: Can I track the budget for my department?**
**A:** If you're a manager or admin, you can view budget reports in the **Reports** section. Regular users see only their own problems.

## üè´ Inventory & Items

### **Q: An item I need to report isn't in the inventory list. What should I do?**
**A:** Contact your administrator to add the item to the inventory system first. Then you can select it when creating your problem report.

### **Q: How do I know which item code to select?**
**A:** Inventory items have names and codes (BRG-001, BRG-002, etc.). Search by item name or look for the code if you know it.

### **Q: Can I report a problem with something that's not in the inventory?**
**A:** The system requires items to be in the inventory first. Contact admin to add missing items to the system.
